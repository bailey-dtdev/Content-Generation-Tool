# Rules suite

Working standards for this repo. These apply to **every** change, in every
phase. The root `CLAUDE.md` points here; the `/verify-rules` skill checks a
diff against them.

| File                    | Governs                                          |
|-------------------------|--------------------------------------------------|
| `documentation.md`      | Markdown hygiene, where docs live, keeping the   |
|                         | build plan current, doc cleanup as we go         |
| `coding-standards.md`   | Code quality, scope discipline, testing          |
| `git-workflow.md`       | Branching, commits, PRs                          |
| `build-process.md`      | How a build-plan phase moves from start to done  |

These rules are deliberately short. If a rule needs a long explanation, the
explanation belongs in the architecture doc, not here.
